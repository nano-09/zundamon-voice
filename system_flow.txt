bot_system_flow
=================
This document outlines the complete system flow of the Zundamon Discord Voice Bot and its Dashboard.

---
[ 1. Dashboard & Ecosystem Startup ]
---
1. The user launches the ecosystem (e.g., via StartZundamon.exe / Launcher.cs) which starts the local servers:
   - Voicevox Engine (http://localhost:50021) for Text-to-Speech generation.
   - Ollama API (http://localhost:11434) for LLM AI interactions.
2. The Dashboard Server (dashboard/server.js) is started on port 3000.
3. The Dashboard automatically spawns the Discord Bot process (`node src/index.js`) locally.
4. The Dashboard listens to standard output/error from the bot, streaming logs, system stats (CPU/RAM), and handling Start/Stop/Restart commands via its frontend web UI.
5. Dashbard pulls data (logs, command usage, guild configs) directly from the Supabase database.

---
[ 2. Bot Initialization ]
---
1. The bot (src/index.js) connects directly to Discord using discord.js and the provided `DISCORD_TOKEN`.
2. It registers its Slash Commands (`/join`, `/leave`, `/config`, `/ai`, etc.) against the Discord API using the `CLIENT_ID`.
3. It loads local dictionaries (e.g., predefined custom emojis, Japanese emoji dictionary).
4. The bot fetches per-guild configurations from the Supabase `guild_configs` table.
5. A Model Context Protocol (MCP) client is initialized to allow the AI module to search the web (DuckDuckGo or Chromium searches).
6. A heartbeat loop begins, returning uptime, ping, and server statistics back to the Dashboard every 5 seconds.

---
[ 3. Guild Management & 2FA Authentication Flow ]
---
1. When the bot joins a new Discord Server (Guild):
   - It captures initial server metadata (Member count, permissions) and seeds it in the Supabase database.
   - The 2FA security system is triggered: The bot generates a 6-digit OTP and emails it to the Bot Owner (`OWNER_EMAIL`).
   - The bot also sends a Direct Message to the Bot Owner on Discord alerting them of the new guild request.
   - The owner must reply to the bot's DM with the 6-digit code.
   - If the code is correct, the bot registers the server in `authorized_guilds.json` and unlocks its functionality.
   - If the server is not authorized, the bot will refuse to play text-to-speech and will block slash command usage.

---
[ 4. Text-To-Speech (TTS) Flow ]
---
1. Users type string messages into the server's configured Text Channel.
2. The bot ignores bots, checking if the guild is authorized.
3. The message is analyzed:
   - Mentions and URLs are sanitized/stripped.
   - Discord Emojis are translated to spoken Japanese representations.
   - The user's display name is optionally prefixed (e.g., "Nanolife。 Message").
   - If a Soundboard keyword is found (from `soundboard.json`), it halts TTS generation and plays an mp3 natively.
4. The raw string is added to the shared playback queue (`src/player.js`).
5. The Player contacts the local Voicevox Engine, requesting a .wav audio buffer of the Zundamon voice.
6. The bot streams the compiled .wav file directly into the guild's current active Voice Channel using `@discordjs/voice`.
7. Activity counters (texts_spoken) are incremented locally.

---
[ 5. AI Assistant Flow ]
---
1. A user mentions the bot, triggering an AI context lookup (`src/ai.js`).
2. The bot retrieves the current server's AI rules/prompt settings and recent message history.
3. If search is required, the MCP server runs a DuckDuckGo search to retrieve contextual web page information.
4. A payload is constructed and routed to the local Ollama API (Llama3 or specified local LLM).
5. Ollama completes the generation and returns text phrasing.
6. The bot optionally pipes the generated response back into the Voicevox Text-to-Speech Engine to read the answer aloud.

---
[ 6. Analytics & Graceful Shutdown ]
---
1. Every 60 minutes, the bot gathers its in-memory counters (voice_minutes, texts_spoken, ais_used) per guild and snapshots them to Supabase `guild_analytics`.
2. The bot logs significant errors or actions to Supabase `logs_v2`.
3. If a shutdown command is received from the Dashboard, the bot smoothly flushes all counters to Supabase, leaves all Active voice channels, and kills its Discord WebSocket.
